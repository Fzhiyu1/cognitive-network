# Hope 记忆外挂

**提出者**：fangzhiyu
**日期**：2026年3月15日
**来源**：[[从记忆理论到状态恢复：meomory原型的设计基础]]

## 定义

主流 LLM（Transformer）是一个没有海马体的大脑皮层——推理能力强，但权重冻结，记不住东西。Hope 记忆外挂的思路是：给冻结的主 LLM 外接一个小型 Hope 模型作为海马体，由它负责记忆的在线学习、匹配和激活。

Hope 模型因为自修改 Titans（DGD 在线更新权重）和 CMS（多频率分层），天然具备真正的记忆能力——推理时权重在变，越用越准。这使它比静态 embedding 匹配更适合做 [[前意识激活层]]。

## 核心要点

- 主 LLM = 皮层（推理、生成，但失忆）；Hope 模型 = 海马体（记忆匹配、在线学习）
- Hope 做记忆匹配比 embedding 相似度强在：权重在线更新（越用越准）、DGD 自带遗忘项（自动压低过时记忆）、CMS 多时间尺度（短期适应 + 长期保留）、能学习用户特有的记忆激活模式
- Hope 记忆模型不需要大——它只做"给定上下文，输出该激活哪些记忆片段"这个窄任务，50-100M 参数足够
- 训练数据来源：先用 embedding 匹配跑 baseline，从实际运行中收集 (context → 哪些记忆被用上了) 的数据，再用这些数据训练 Hope 模型
- 直接映射知识库概念：Hope 记忆模块 = [[关联记忆]]，DGD 遗忘项 = [[主动遗忘]]，CMS = [[多层有损压缩]]
- 这是中间态架构——终态是主模型本身具备 Hope 机制，不再需要外挂。但在此过程中积累的压缩/分层/线索匹配经验仍然有价值

## 实现路径

1. 从 OpenClaw session 压缩出分层记忆片段（L0/L1/L2）
2. 用 embedding 匹配做简单激活层先跑起来（baseline）
3. 从运行中收集 (context, activated_memory) 配对数据
4. 用配对数据训练小 Hope 模型替换 embedding 匹配层
