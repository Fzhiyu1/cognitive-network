# 记忆与图书馆

**提出者**：fangzhiyu
**日期**：2026年3月15日
**来源**：[[从记忆理论到状态恢复：meomory原型的设计基础]]

## 定义

向量库（RAG）本质上是图书馆：无损存储、相似度搜索、不学习、不遗忘。关联记忆 M(key)→value 更接近真正的记忆：有损压缩、O(1) 激活、能学习、能遗忘、能泛化。两者的核心区别在于**压缩**——图书馆独立存放每个条目，记忆把所有知识压进同一组权重。

判断标准：如果你得去搜索才能找到，那是图书馆；如果线索出现时它自己浮现，那是记忆。

## 核心对比

| | 向量库（RAG） | 关联记忆 M(key)→value | Engram 哈希查找 |
|--|-------------|---------------------|---------------|
| 存储 | 每个块一个向量，独立存放 | 所有知识压缩进权重矩阵 | 每个 N-gram 一个 embedding 槽位 |
| 检索 | 相似度搜索 O(n)/O(log n) | 矩阵乘 O(1) | 哈希映射 O(1) |
| 学习 | 不学 | DGD 在线更新 | 不学（训练后冻结） |
| 遗忘 | 不忘，要手动删 | DGD 遗忘项自然衰减 | 不忘 |
| 泛化 | 不泛化，返回原始块 | 能泛化，没见过的 key 也能产出合理 value | 不泛化，未命中就是空 |
| 容量 | 加硬盘就能扩 | 矩阵维度固定，有限 | 表可以很大（100B+） |

## 工程启发：三种机制叠加

三种检索机制不是互斥的，可以分层叠加，越往上越像"记忆"，越往下越像"图书馆"：

- **L1 关联记忆**（DGD/CMS）：O(1)，能学习能遗忘，捕捉用户在特定上下文下的记忆激活模式
- **L1.5 哈希查找**（Engram 式）：O(1)，静态但大容量，存稳定的实体知识和固定模式
- **L2 向量搜索**（RAG）：O(log n)，不学不忘但什么都存着，关联记忆没命中时的 fallback

## 关联概念

- [[关联记忆]] — M(key)→value 是记忆的基本形式，向量库不是
- [[条件记忆]] — Engram 介于两者之间：O(1) 查找但不学习
- [[多层有损压缩]] — 关联记忆的有损性是它能泛化和遗忘的根源
- [[主动遗忘]] — 向量库无自然遗忘机制，关联记忆有
- [[前意识激活层]] — O(1) 的关联记忆和哈希查找天然适合做前意识激活
- [[记忆假肢]] — 向量库是最"假肢"的假肢（纯图书馆），关联记忆是更接近真记忆的假肢
