# 缓存放大工作流

**提出者**：fangzhiyu
**日期**：2026年3月11日
**来源**：[[从Token轨迹到工作状态：AI原生工作流的三口径观察]]

## 定义

缓存放大工作流指的是：在长上下文和 agent 协作环境里，系统通过反复复用既有上下文，把少量新增指令接到一个持续展开的大任务线程上，进而让总 token 处理量远高于用户显式新增的输入输出量。

## 核心要点

- 高 cache 不等于作弊，而是说明上下文被持续复用
- cache 越高，越可能意味着任务主线连续、agent 在接续推进，而不是每次都从零开始
- 这种工作流的规模，不能只看新增输入输出，还要看上下文复用所带来的总吞吐
- 高 cache 比例常常伴随少数超大 session、强任务连续性和较少的上下文重建成本
- cache 因而可以成为判断执行态和工作流连续性的一个行为学信号
