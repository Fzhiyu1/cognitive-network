# 从记忆理论到状态恢复：meomory 原型的设计基础

**讨论者**：fangzhiyu & Claude Opus 4.6
**日期**：2026年3月15日
**背景**：用户计划在 OpenClaw 的运行数据上构建一个记忆系统原型，在动手之前，先回到知识库中的记忆理论进行审视和补充，确定设计基础。

---

## 一、理论审计：八个核心概念能否作为基底

对话首先回顾了知识库中已有的八个记忆概念（[[可行动状态]]、[[关联记忆]]、[[多层有损压缩]]、[[主动遗忘]]、[[外置工作记忆]]、[[知识库元认知]]、[[缓存放大工作流]]、[[知道与推测不分]]），逐一评估其作为工程基底的可靠性。

站得稳的部分：记忆 = 为行动服务的压缩、遗忘是压缩代价、主动遗忘 = 检索控制、外置工作记忆作为工程模式。

需要警惕的部分：[[关联记忆]] 的统一性可能过度光滑——MLP 权重与人类情景记忆共享"映射"属性，但更新机制、上下文敏感性、组合性差异巨大。类比不等于同构。

理论尚未回答的三个工程问题：检索机制（什么时候取出什么）、记忆粒度（一条记忆的边界是什么）、冲突解决（新旧记忆打架怎么办）。

结论：可以作为基底，但这些缺口需要在构建过程中回答。

## 二、分层设计：L0 是离行动最近的层

最初的直觉是把 L0 定义为"原始事件流"，但用户指出 L0 应该直接进上下文、支撑当前决策。这把分层方向反转了，对齐到 CPU 缓存的命名逻辑：

| 层 | 定位 | 检索成本 |
|----|------|---------|
| L0 | 自动注入上下文，零检索成本 | 无 |
| L1 | 按需激活的片段 | 一次 read |
| L2 | 原始 session 存档 | 高 |

压缩方向是 L2 → L1 → L0。L0 是最压缩、最可行动的层。

## 三、被动激活 vs 主动搜索：记忆不是图书馆

关键转折点：如果 AI 必须"决定去读"才能获得记忆，那就是图书馆，不是记忆。人类记忆的默认模式是线索驱动的自动提取——一个线索出现，相关内容浮现，意识没有参与这个决策。

从你的视角看是被动的，从机制的视角看是主动的。神经网络的扩散激活发生在意识决策之前。

这意味着系统需要一个 LLM 看不见的**前意识激活层**：消息到达 → 线索匹配 → 命中的记忆被注入上下文 → LLM 开始思考时记忆已经"在那里"了。OpenClaw 的 gateway 架构天然有空间做这件事。

## 四、真正的记忆 = 权重修改

最深的一层追问：如果真正的记忆只有权重修改才能实现，那我们在做的一切算什么？

答案：[[记忆假肢]]。

我们在做的所有事情——L0、L1、片段、cue、注入——都不是记忆。每次对话，LLM 都是一个失忆的人被推进房间，手里塞了一叠便条。便条拿走，它什么都不知道。

但人类自己也大量依赖记忆假肢：语言、文字、仪式、知识库。假肢不是记忆，但假肢确实让系统做到了单靠生物/参数记忆做不到的事。

更准确的定位：**我们不是在让 AI 记住，而是在每次失忆之后，用最低成本把它恢复到最接近"记得"的 [[可行动状态]]。** 这是一个状态恢复系统，不是记忆系统。

## 五、文本层的联结问题

记忆片段的存在形式不应该是文档，而应该是足够小的 fragment。联结不靠显式链接，靠两个机制：

1. **共享词表**：片段间使用一致的术语，LLM 的注意力在处理时自动建立关联
2. **共同加载**：哪些片段被同时放入上下文，决定了哪些记忆被关联——LLM 的注意力机制本身就是 [[关联记忆]] 硬件

这意味着压缩时必须保持术语一致性，而 L0 片段必须足够小，小到一次能装够多个，让关联有空间发生。

---

*这次探索的核心发现是：我们构建的不是记忆系统，而是状态恢复系统——一个记忆假肢。知道这个边界，才能正确设定设计目标：不追求让 AI "记住"，而是让它每次醒来后尽快回到能继续行动的位置。*
