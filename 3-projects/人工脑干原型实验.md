# 人工脑干原型实验

**状态**：准备阶段
**来源**：[[人工脑干：从目标真空到RL+LLM双系统架构]]

## 目标

在Minecraft环境中验证"脑干层"的可行性 — 一个监控生存指标、能覆盖皮层决策的模块。

## 核心类比：给老鼠装人脑

- **老鼠身体**（执行层）：有固定的本能动作（跑、躲、觅食），不学新动作，但现有能力足够生存
- **人脑**（LLM皮层）：利用老鼠已有的本能做更聪明的决策（什么时候跑、往哪跑、先吃还是先躲）
- **脑干**（待建）：监控生存指标，必要时覆盖皮层决策

## 实现路线

### Phase A：Mindcraft路线（当前）

基于Mindcraft框架快速验证脑干逻辑。Mindcraft已解决LLM↔Minecraft通信，提供预定义动作原语（`!attack`、`!consume`、`!moveAway`等）和双通道感知（API数值 + 可选视觉截图）。脑干层插入LLM命令输出和Mineflayer执行之间。

优势：macOS可跑，无需GPU，Node.js生态
局限：感知基于API（非端到端视觉），动作是预定义命令

### Phase B：JarvisVLA路线（远期）

将执行层从Mineflayer替换为JarvisVLA，获得端到端视觉感知和动作控制。脑干层和技能缓存设计两条路线通用。

优势：感知真实（视觉+语义→动作），更接近认知架构终态
局限：需要Linux + GPU

### 架构说明

- 执行层有两条路线：Phase A用Mindcraft/Mineflayer（命令式，易上手），Phase B换JarvisVLA（端到端视觉，更真实）
- 技能缓存是[[技能编译]]的符号化中间态，用Voyager式代码库绕开[[连续基质问题]]
- 脑干层必须快于皮层反应（[[行动先于意识]]），LLM脑干只是探索阶段的临时方案

## 第一阶段：Mindcraft + LLM脑干

基于Mindcraft框架，用LLM临时扮演脑干，探索脑干需要做什么。

### 环境
- Minecraft Java版（Mindcraft支持到1.21.6）
- 生存模式，难度正常
- Node.js + Mineflayer

### 脑干插入点
```
Mindcraft原始流程：LLM → 命令 → Mineflayer执行
加入脑干后：    LLM → 命令 → [脑干检查] → Mineflayer执行
```

脑干输入从Mineflayer API直接读取：`bot.health`、`bot.food`、`bot.nearestEntity()`、`bot.time.timeOfDay`

### 分工
- **皮层LLM**：接收状态摘要+任务目标，输出Mindcraft命令（`!attack`、`!consume`等）
- **脑干LLM**：每tick接收生存指标，判断是否需要覆盖皮层决策
- **Mineflayer**：接收最终命令，执行游戏操作

### 观察指标
- 脑干在什么条件下发出中断
- 皮层被中断后的行为变化
- 是否涌现出节律性生存行为（天黑找掩体、定期进食）
- 脑干的决策模式能否归纳为有限规则

## 第二阶段：蒸馏到小模型

记录LLM脑干的所有（输入状态→决策）数据，用监督学习蒸馏到小型分类器/MLP。脑干本质是分类任务（数值输入→有限动作类别），不需要语言能力，小模型足矣。可跳过"规则提炼"直接蒸馏。

## 第三阶段：技能缓存 + 持续优化

引入Voyager式符号化技能库，缓存皮层的成功决策模式。观察哪些技能能被符号化、哪些不能，为未来的神经级[[技能编译]]划定边界。如果用RL优化脑干，LLM皮层可反向加速训练（写奖励函数、设计课程、提供示范轨迹 — 康复训练类比）。

## 远期扩展：皮层-脑干双向竞争

当前架构只有单向中断（脑干覆盖皮层）。更完整的模型应包含反向：皮层否决脑干（如脑干提议进食，皮层判断有危险而压制）。

近似方案：给脑干动作加执行延迟窗口，紧急动作直接执行，非紧急动作等待皮层否决。对应Libet的"自由否决"概念（[[行动先于意识]]）。

此机制会显著增加系统复杂度，留到基础架构验证后再引入。
