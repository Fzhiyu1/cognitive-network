# meomory 记忆假肢原型

**状态**：设计完成，准备开始阶段一
**来源**：[[从记忆理论到状态恢复：meomory原型的设计基础]]

## 目标

给已有的知识库和 OpenClaw 系统补上**被动激活**能力——从"需要手动翻的笔记本"变成"自动浮现的记忆"。不是从零造新系统，是在已有基础设施上加一条缺失的腿。

本质上这不是记忆系统，是 [[记忆假肢]] / 状态恢复系统：在每次 LLM 失忆后，用最低成本恢复到 [[可行动状态]]。

## 核心架构

### 记忆分层

| 层 | 定位 | 检索成本 | 机制 | 阶段 |
|----|------|---------|------|------|
| L0 | 永远在上下文（SOUL.md 等） | 零 | 自动注入 | 已有 |
| L1 | [[前意识激活层]] 自动匹配注入 | 接近零 | [[关联记忆]] DGD/CMS（能学能忘） | 三/四 |
| L1.5 | 稳定知识快速查找 | O(1) | [[条件记忆]] Engram 式 N-gram 哈希 | 未来 |
| L2 | 兜底搜索 | O(log n) | 向量相似度（qwen3-embedding） | 一 |
| L3 | 冷存储 | 高 | 原始 session 存档 | 一 |

### 消息流

```
用户消息
  ↓
[前意识激活层: 消息 → cue 提取 → 记忆匹配 → 注入]
  ↓
LLM thinking（可调 recall_memory 工具做主动召回）
  ↓
LLM 回复
  ↓
thinking 内容 → 下一轮的深层 cue
```

### 五个设计原则

1. **被动激活优于主动搜索**：LLM 不需要决定"要不要读"，记忆在消息到达前已注入
2. **文本层联结靠共享词表 + 共同加载**：不靠显式链接，靠术语一致性 + 注意力自动关联
3. **三种检索机制分层叠加**（[[记忆与图书馆]]）：越往上越像记忆（关联记忆），越往下越像图书馆（向量搜索）
4. **Thinking 是更好的 cue 来源**：用上一轮 thinking 为当前轮预取深层记忆
5. **20-25% 上下文预算**（参考 Engram 分配法则）：注入记忆占上下文不超过 1/4

```
阶段一（图书馆）         阶段二（智能图书馆）       阶段三（会学习）         阶段四（多时间尺度）

  → L1/L0 片段           → 高质量 L1/L0

cosine 匹配             cosine + 门控过滤         DGD 关联记忆          CMS 三层
  → 等权注入              → 加权注入                → 在线学习            → 短/中/长期
                         + thinking cue             + 训练对收集          + 分频更新

检索: O(n)              检索: O(n) + 过滤         检索: O(1) 矩阵乘     检索: O(1) × 3 层

```

## 理论基础

本项目的设计决策来自知识库中的记忆理论网络：

- [[可行动状态]] — 记忆的终点是"现在能怎么做"，不是"知道了什么"
- [[关联记忆]] — M(key)→value，DGD/CMS 是工程实现
- [[多层有损压缩]] — 压缩必然有损，遗忘是代价不是故障
- [[主动遗忘]] — 检索控制，不是删除。上下文门控是推理时的实现
- [[外置工作记忆]] — 压缩、纠偏、回放三个核心操作
- [[记忆假肢]] — 我们做的是假肢，不是真记忆，但假肢有实用价值
- [[前意识激活层]] — 被动激活，LLM 不需要决定"要不要读"
- [[条件记忆]] — Engram 从工程路径独立验证了同一套架构
- [[Hope 记忆外挂]] — DGD+CMS 零件用于在线学习的激活层
- [[记忆与图书馆]] — 向量库是图书馆，关联记忆才接近真正的记忆
- [[知识库元认知]] — 知识库本身已经是运行中的记忆假肢
