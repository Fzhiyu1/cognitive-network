# meomory 记忆假肢原型

**状态**：构想阶段
**来源**：[[从记忆理论到状态恢复：meomory原型的设计基础]]
**工作目录**：`/Users/fangzhiyu/run/meomory`

## 目标

给已有的知识库和 OpenClaw 系统补上**被动激活**能力——从"需要手动翻的笔记本"变成"自动浮现的记忆"。不是从零造新系统，是在已有基础设施上加一条缺失的腿。

本质上这不是记忆系统，是 [[记忆假肢]] / 状态恢复系统：在每次 LLM 失忆后，用最低成本恢复到 [[可行动状态]]。

## 核心设计决策

1. **记忆分层（L0 离行动最近）**

| 层 | 定位 | 检索成本 | 机制 |
|----|------|---------|------|
| L0 | 永远在上下文（SOUL.md 等） | 零 | 自动注入 |
| L1 | [[前意识激活层]] 自动匹配注入 | 接近零 | [[关联记忆]] DGD/CMS（能学能忘） |
| L1.5 | 稳定知识快速查找 | O(1) | [[条件记忆]] Engram 式 N-gram 哈希 |
| L2 | 兜底搜索 | O(log n) | 向量相似度（qwen3-embedding） |
| L3 | 冷存储 | 高 | 原始 session 存档 |

2. **被动激活优于主动搜索**：[[前意识激活层]] 在 LLM 之前运行，LLM 不需要决定"要不要读"，记忆已经在上下文里了
3. **文本层联结靠共享词表 + 共同加载**：不靠显式链接，靠术语一致性 + LLM 注意力在共现片段间自动关联
4. **三种检索机制分层叠加**（[[记忆与图书馆]]）：关联记忆（能学能忘）→ 哈希查找（静态大容量）→ 向量搜索（兜底），越往上越像记忆，越往下越像图书馆
5. **Thinking 作为深层 cue 来源**：用上一轮 thinking 内容为当前轮预取记忆；同时注册 `recall_memory` 工具让模型可主动召回

## 消息流架构

```
用户消息
  ↓
[第一轮激活：消息 → cue 提取 → 记忆匹配 → 注入]
  ↓
LLM thinking（可调 recall_memory 工具做主动召回）
  ↓
LLM 回复
  ↓
thinking 内容存入 → 下一轮的深层 cue 来源
```

## 数据源

- OpenClaw sessions：53 活跃 + 49 已删除，~31MB JSONL
- 每个 session 是完整交互时间线（消息、工具调用、工具结果、模型切换）
- 运行日志（/tmp/openclaw/）信息密度低，优先用 session

## 实施路径

### 阶段一：跑通管线（不需要学新东西）

- [ ] 从 OpenClaw session 压缩出 L0/L1/L2 片段（文本处理，不需要 ML）
- [ ] 用 qwen3-embedding 做 cosine 匹配（向量库版 baseline）
- [ ] 在 OpenClaw gateway 加中间件注入匹配结果
- [ ] 给模型注册 recall_memory 工具
- 产出：一个能跑的"图书馆版"记忆系统

### 阶段二：从图书馆升级成记忆

- [ ] 加上下文门控（匹配结果按相关度加权，不等权注入）
- [ ] 加频率统计（Zipfian 分层，高频命中片段提升到 L0）
- [ ] 用上一轮 thinking 做下一轮的深层 cue
- 产出：有门控、有分层、有扩散激活

### 阶段三：加在线学习

- [ ] 实现 DGD 更新规则替换 cosine 匹配做 L1 激活
- [ ] 收集 (context, activated_memory) 训练对
- 产出：匹配器能从经验中学习，越用越准

### 阶段四：加多时间尺度

- [ ] 加 CMS 多频率层（高频：本次对话 / 中频：近期重心 / 低频：长期模式）
- 产出：完整的可在线学习记忆模块

## 理论基础

本项目的设计决策来自知识库中的记忆理论网络：

- [[可行动状态]] — 记忆的终点是"现在能怎么做"，不是"知道了什么"
- [[关联记忆]] — M(key)→value，DGD/CMS 是工程实现
- [[多层有损压缩]] — 压缩必然有损，遗忘是代价不是故障
- [[主动遗忘]] — 检索控制，不是删除。上下文门控是推理时的实现
- [[外置工作记忆]] — 压缩、纠偏、回放三个核心操作
- [[记忆假肢]] — 我们做的是假肢，不是真记忆，但假肢有实用价值
- [[前意识激活层]] — 被动激活，LLM 不需要决定"要不要读"
- [[条件记忆]] — Engram 从工程路径独立验证了同一套架构
- [[Hope 记忆外挂]] — DGD+CMS 零件用于在线学习的激活层
- [[记忆与图书馆]] — 向量库是图书馆，关联记忆才接近真正的记忆
- [[知识库元认知]] — 知识库本身已经是运行中的记忆假肢
